Class Object

Represents any entity in the ldap server

Any object returned by lookup operations will be of this class or will inherit from it. It contains all the object's data, represented by instances of the ADX\Core\Attribute class.

ADX\Core\Object implements ArrayAccess, Iterator, Countable, JsonSerializable uses ADX\Core\Jsonizer

public static ADX\Core\Object
# read( string $dnOrFilter, array $attributes, ADX\Core\Link $adxLink )

Read a single object from the server

Read a single object from the server

Use this method to read the specified attributes of a single entity in the directory server if you know the distinguishe name of the object, or if you can provide a search filter that returns exactly zero or one object.

Example:

use ADX\Core\Object;            // Import the class into the current namespace

// Read the object with specified distinguished name
$user = Object::read(   "CN=Karen Berge,OU=admins,DC=corp,DC=company,DC=com",
                        ['name', 'mail'],
                        $link );                // $link is a fully configured ADX\Core\Link object

// Read an object specified by the search filter
try
{
        $user = Object::read( "(&(samaccountname=jsmith))", ['name', 'mail'], $link );
}
catch ( ADX\Core\Exception $e )
{
        // More than one object found - refine your search query!
        echo $e->getMessage();
        exit;
}

// Use the object somehow...
echo $user->mail->value(0);

# restore( string $json, ADX\Core\Link $adxLink )

Restore an Object from a json string

Restore an Object from a json string

If you previously jsonized an instance of the ADX\Core\Object class, you can convert it to an object again using this method.

All data that the json string holds will be considered as locally modified! That means, if you call ADX\Core\Object::update() on a restored object, all data will be sent to the server. As such, it is recommended that you use this method only with modified data if you plan to call ADX\Core\Object::update().


This method is currently experimental.

public ADX\Core\Object
# __construct( ADX\Core\Link $adxLink, array $attributes = array() )

Create a brand new object to be potentially saved to the server later on

Create a brand new object to be potentially saved to the server later on

Create a new object, modify its attributes and optionally store the object on the directory server.

Example:

use ADX\Core\Object;    // Import the class into the current namespace

// Create the instance and specify the 'objectclass', 'cn' and 'samAccountName'
// attributes immediately
// ( $link is an instance of fully configured ADX\Core\Link )
$object = new Object( $link, ['objectclass' => 'user', 'cn' => 'Rossmann Robert', 'samAccountName' => 'robertrossmann'] );

// Optionally, further modify the object as needed
$object->givenName->set( 'Robert' );
$object->sn->set( 'Rossmann' );

// Save the object to the server
$object->create( 'OU=admins,DC=corp,DC=company,DC=com' );

To create users in Active Directory, you should use the ADX\Classes\User class for best usability. The axample above is just to demonstrate the basic usage.

Get or set a bit in a bitfield attribute

Get or set a bit in a bitfield attribute

This method is useful for getting or setting bits in a bitfield attributes like userAccountControl, systemFlags etc.

Example:

// $link is a valid connection to a directory server

public ADX\Core\Result
# resolve( string $attribute, string|array $attributes = null )

Resolve the attribute into individual instances of the ADX\Core\Object class

Resolve the attribute into individual instances of the ADX\Core\Object class

If the attribute holds distinguished name(s) as its values, it is considered to be resolvable. Resolving such attribute means that the distinguished names the attribute holds will be read from the directory server ( using ADX\Core\Object::read() ) and turned into editable instances of the Object class. You can supply an array of attributes that these objets should have when loaded from server.

This method is very handy for retrieving information ( email address, name etc.) about e.g. members of a group.

You must load the property from the server first in order to resolve it.

Example:

use ADX\Core\Object;    // Import the class into the current namespace
// $link is a fully configured and bound ADX\Core\Link object

// Read the list of members of a group
$group = Object::read(
        'CN=My Security Group,OU=Groups,DC=corp,DC=company,DC=com',
        ['member'],
        $link );

// Resolve the group's members into individual
// objects ( each object will have the 'mail' attribute loaded )
$members = $group->resolve( 'member', ['mail'] );

// You have two options now.
// No. 1: The $members is an instance of the ADX\Core\Result class -
// you can utilise it's capabilities to traverse through the objects
// in the resultset
$members->each( function( $object )
{
        print_r( $object->mail->value() );
});

// No. 2: Access those individual objects from the $group,
// as if it were just another attribute
print_r( $group->member->value(0)->mail->value() ); // Prints the email address of the first object in the 'member' attribute
print_r( $group->member(0)->mail() );           // Use a shorter syntax to print the same information
