Class User

Class representing a 'user' objectClass in Active Directory

This class provides user-class-specific methods to simplify user creation and manipulation in Active Directory. It also provides some MS Exchange-related functionality like managing mailboxes and mail users.

Create a new instance of the user

Create a new instance of the user

In order to successfully create a new user account you must provide the following attributes the moment you instantiate a new object:

  • cn - the user's cn forms part of the distinguished name and cannot be omitted
  • samAccountName - the username used to log in


The following attributes are calculated automatically for you:

  • name, displayName - equals to cn
  • userPrincipalName - equals to samAccountName + @ + domain name ( domain name is generated from rootDSE's defaultNamingContext )
  • userAccountControl - if you provide user's password at this moment, it will include only the ADX\Enums\UAC::NormalAccount bit; if no password is present, it will also include ADX\Enums\UAC::PasswdNotReqd and ADX\Enums\UAC::AccountDisable bits. This is default behaviour of Active Directory.

If you set the user's password before creating the account, it will be created without the requirement to reset the password upon next logon. To force user to change the password next time he/she logs in, use the ADX\Classes\User::force_password_change() method before calling ADX\Classes\User::create().

# create( string $dn )

Create the user on the server

Create the user on the server

This method provides some extra functionality to the standard ADX\Core\Object::create() method.

If you have set the user's password then the following updates will be made to the user account:

  • The ADX\Enums\UAC::PasswdNotReqd bit will be turned off
  • The user account will be created as enabled
  • The user will not be forced to change the password unless you have explicitly forced a password change with ADX\Classes\User::force_password_change()


If, for whatever reason you need any of the above userAccountControl bits to be different you must override the values after the user has been created on the server, meaning, after you have called the create() method on the object.
